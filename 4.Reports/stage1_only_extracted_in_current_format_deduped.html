Stage 1

공통 backbone 잠금

🧭 이 Stage가 하는 일

post hoc drift를 막는 공통 기준 고정

  • event definition, dataset split, horizon choice, threshold choice, formula choice, interpretation hierarchy를 먼저 확정합니다.
  • 특히 1–10년 horizon vector와 threshold grid를 고정해 이후 KM, no-cure, frequentist cure, Bayesian cure 결과를 같은 ruler로 비교할 수 있게 합니다.
  • 이 Stage는 model-fitting stage가 아니라, 전체 workflow의 backbone lock 단계입니다.
🗂️ 필수 산출물

downstream stages가 그대로 상속해야 하는 registry

  • dataset / event / remission / censoring count 요약
  • person-time summary와 age / sex / site summary
  • modeling registry, formula registry, horizon registry, threshold registry, metadata registry
  • saved scaling constants와 이후 Stage에 넘길 analysis datasets
🔒 왜 중요한가

같은 질문을 같은 축에서 비교하기 위한 준비

Stage 1이 흔들리면 뒤 Stage의 차이가 모델 차이인지, 아니면 기준 변경 때문인지 분리할 수 없습니다.

따라서 이 Stage는 결과 자체를 내는 단계라기보다, Stage 2–10의 해석 가능성과 직접 비교 가능성을 보장하는 기반 단계입니다.
