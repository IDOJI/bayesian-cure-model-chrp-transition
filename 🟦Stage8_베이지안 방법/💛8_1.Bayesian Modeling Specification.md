# ****Stage 8. Bayesian mixture-cure model as a substantive-prior, stabilization, and uncertainty block**** 🌌

  

  

---

  

## ****0. Role and scope of Stage 8**** 🔒

  

This stage is ****not**** a replacement for the earlier workflow.  

It is the ****Bayesian cure block only****.

  

It assumes that the earlier stages have already been completed, including:

  

- follow-up maturity description,

- timing-difference assessment,

- cure-appropriateness screening.

  

It also does ****not**** replace the later unified comparison stage.

  

### ****The role of Stage 8 is fourfold****

  

1. ****stabilize**** cure-model estimation when MLE is unstable,

2. ****carefully encode external incidence information**** in the incidence component only,

3. ****propagate posterior uncertainty**** more transparently,

4. ****export Bayesian predictions and threshold-based outputs**** on the same horizon/threshold grid used later for direct comparison with ****KM****, ****no-cure****, and ****cure MLE**** models.

  

### ****Important boundary****

  

- ****Stage 8 main likelihood**** keeps the project’s common event rule:  

  ****transition = event****, ****remission = censoring****.

- ****Stage 9**** is where remission is reanalyzed as a competing event / multi-state sensitivity, especially for ****PNU****.

- ****Stage 10**** is where the final direct comparison among ****KM / no-cure / cure MLE / Bayesian cure**** is made on the same horizons and thresholds.

- Use `days_followup` as the default time variable.

- Treat `status_num %in% c(0, 2)` as censoring in the ****main**** analysis.

# ****2. Data and event definition**** ⏱️

  

For subject \(i\), define:

  

\[

\delta_i = I(\text{status\_num}__i = 1)_

_\]_

  

\[

T_i = \max\!\left(\frac{\text{days\_followup}__i}{365.25}, 10^{-8}\right)_

_\]_

  

Interpretation:

  

- `status_num = 1` → transition event

- `status_num ∈ {0, 2}` → censoring in the ****main Bayesian likelihood****

  

### ****Stage-8 handling of remission****

  

In the ****main Stage-8 Bayesian block****, remission is treated as censoring to preserve comparability with the common project backbone.  

A remission-sensitive reanalysis is ****not**** part of this likelihood and is delegated to ****Stage 9****.

# ****3. Mixture-cure decomposition**** 🧩

  

The population survival function is

  

\[

S__{\text{pop}, i}(t) = (1 - \pi_i) + \pi_i S_u(t \mid i)_

_\]_

  

where:

  

- \(\pi_i\): probability that subject \(i\) belongs to the ****susceptible / eventually transition-capable**** subgroup

- \(1 - \pi_i\): ****non-susceptible / cure-like**** proportion

- \(S_u(t \mid i)\): survival in the susceptible subgroup

  

The population risk is

  

\[

R_i(t) = 1 - S__{\text{pop}, i}(t) = \pi_i \{1 - S_u(t \mid i)\}_

_\]_

  

This keeps the Bayesian block directly aligned with the cure estimand already used elsewhere in the workflow.

X_i^{inc} =

Let

  

\[

a_i = age__{s, i}_

_\]_

  

that is, the already-saved scaled age from common preprocessing:

  

\[

age_s = \frac{age\_exact\_entry - \operatorname{mean}(age\_exact\_entry)}{2 \cdot \operatorname{sd}(age\_exact\_entry)}

az_i = a_i z_i

This follows the project-wide scaling logic: numeric predictors are standardized on a ****2-SD scale****, whereas binary inputs stay on their natural scale. This makes coefficient magnitudes more comparable and improves regularized prior specification.

# ****5. Incidence component**** 📍

  

## ****5.1 External evidence rule****

  

Only ****one substantive external source**** is used here:

  

- ****van der Werf et al.****-type age–sex incidence evidence

  

This external information is used ****only in the incidence component****, and only as a ****prior-centering device****, not as a hard truth.  

It does ****not**** constrain latency directly, and it is ****not**** treated as a direct prior on the cure fraction itself.

  

---

  

## ****5.2 Anchor transformation****

  

For each age–sex stratum \(g\), define:

  

\[

r_g = \frac{r_g^{(10k)}}{10000}, \qquad q_g = 1 - e^{-r_g}, \qquad y_g = \operatorname{logit}(q_g)

\]

  

Use the Female <20 general-population anchor:

  

\[

\alpha__{GP}^{vdW} = \operatorname{logit}\!\left(1 - e^{-0.69/10000}\right) = -9.581369553169_

_\]_

  

---

  

## ****5.3 Prior mean vector****

  

Define:

  

\[

\mu_M = y__{M,_<20} - y_{F,<20}

_\]_

  

\[

\mu__{20} = y__{F,20\text{–}29} - y__{F,_<20}

_\]_

  

\[

\mu__{30} = y__{F,30\text{–}39} - y__{F,_<20}

_\]_

  

\[

\mu__{M20} = \big(y__{M,20\text{–}29} - y__{M,_<20}\big) - \big(y_{F,20\text{–}29} - y_{F,<20}\big)

_\]_

  

\[

\mu__{M30} = \big(y__{M,30\text{–}39} - y__{M,_<20}\big) - \big(y_{F,30\text{–}39} - y_{F,<20}\big)

_\]_

  

Thus,

  

\[

\mu^{vdW} = (0.419871845822,\ 0.907608052926,\ 0.586202561451,\ 0.466865123863,\ 0.037997248763)^\top

\]

  

---

  

## ****5.4 Prior scale vector****

  

Using the previously derived interval-based prior scales,

  

\[

s_0^{vdW} = (0.132789397422,\ 0.173731076538,\ 0.191221553945,\ 0.270393197518,\ 0.302838606651)^\top

\]

## ****5.5 Main incidence regression****

  

### ****Default Stage-8 incidence model****

  

\[

\operatorname{logit}(\pi_i) = \alpha__{GP}^{vdW} + \Delta + X_i^{inc}\beta_

_\]_

  

with priors

  

\[

\Delta \sim N(0, 3.5^2)

\]

  

\[

\beta_j \sim N(\mu_j^{vdW}, s__{0,j}^2), \qquad j = 1, \dots, 5_

_\]_

  

Interpretation:

  

- \(\Delta\) absorbs cohort-specific baseline departure from the general-population anchor

- \(\beta\) carries the age-band × sex pattern, softly centered by the external evidence

  

### ****Merged supplementary incidence-site branch****

  

For a ****supplementary merged branch only****,

  

\[

\operatorname{logit}(\pi_i) = \alpha__{GP}^{vdW} + \Delta + X_i^{inc}\beta + \beta_s^{inc} s_i_

_\]_

  

\[

\beta_s^{inc} \sim N(0, 0.5^2)

\]

  

### ****Locked rule****

  

In Stage 8, ****sex × age interaction is fixed in the incidence part****.  

It is ****not**** turned on/off there, because that interaction is exactly the structure used to encode the external prior.  

The on/off structural switching happens in the ****latency**** part, not in this incidence part.

  

### ****Incidence link****

  

Use ****logit**** as the default incidence link.  

Alternative links such as probit or cloglog are ****not**** part of the main Stage-8 grid and can be considered only as later sensitivity analyses.

# ****6. Latency component**** ⌛

  

## ****6.1 Median-based parameterization****

  

For all latency families,

  

\[

\mu_i^{lat} = \gamma_0 + X_i^{lat}\gamma

\]

  

\[

m_i = \exp(\mu_i^{lat})

\]

  

Interpret \(m_i\) as the ****susceptible-subgroup median transition time****.

## ****6.2 Latency structural branches****

  

### ****PNU / SNU****

  

`L0`:

  

\[

\mu_i^{lat} = \gamma_0 + \gamma_a a_i + \gamma_m z_i

\]

  

`L1`:

  

\[

\mu_i^{lat} = \gamma_0 + \gamma_a a_i + \gamma_m z_i + \gamma__{am} a_i z_i_

_\]_

  

### ****merged****

  

`L0S0`:

  

\[

\mu_i^{lat} = \gamma_0 + \gamma_a a_i + \gamma_m z_i

\]

  

`L1S0`:

  

\[

\mu_i^{lat} = \gamma_0 + \gamma_a a_i + \gamma_m z_i + \gamma__{am} a_i z_i_

_\]_

  

`L0S1`:

  

\[

\mu_i^{lat} = \gamma_0 + \gamma_a a_i + \gamma_m z_i + \gamma_s s_i

\]

  

`L1S1`:

  

\[

\mu_i^{lat} = \gamma_0 + \gamma_a a_i + \gamma_m z_i + \gamma__{am} a_i z_i + \gamma_s s_i_

### ****Locked rule for merged****

  

At the current stage, keep both:

  

- ****site-free merged models****

- ****site-adjusted merged models****

The ****incidence-site term**** remains a ****supplementary branch****, not the default merged incidence structure.

## ****6.3 Priors for latency coefficients**** ✍️

  

\[

\gamma_0 \sim N(0, 2.5^2)

\]

  

\[

\gamma_j \sim N(0, 1^2), \qquad j \in \{a, m, am, s\}

\]

  

Rationale:

  

- \(a_i = age_s\) is already on a 2-SD scale,

- binary covariates remain on their natural scale,

- this makes a common weakly informative prior scale reasonable,

- and provides regularization precisely where MLE instability is a concern.

# ****7. Latency family definitions**** 📚

  

## ****7.1 Exponential****

  

Use a median-based scale parameterization:

  

\[

\lambda_i = \frac{m_i}{\log 2}

\]

  

\[

S_u(t \mid i) = \exp(-t / \lambda_i)

\]

  

\[

f_u(t \mid i) = \lambda_i^{-1} \exp(-t / \lambda_i)

\]

  

No shape parameter.

  

---

  

## ****7.2 Weibull****

  

\[

k_W = \exp(\rho_W)

\]

  

\[

\lambda_i = \frac{m_i}{(\log 2)^{1/k_W}}

\]

  

\[

S_u(t \mid i) = \exp\!\left[-\left(\frac{t}{\lambda_i}\right)^{k_W}\right]

\]

  

\[

f_u(t \mid i) = \frac{k_W}{\lambda_i} \left(\frac{t}{\lambda_i}\right)^{k_W - 1} \exp\!\left[-\left(\frac{t}{\lambda_i}\right)^{k_W}\right]

\]

  

---

  

## ****7.3 Log-normal****

  

\[

\log T_i \mid (U_i = 1) \sim N(\mu_i^{lat}, \sigma__{LN}^2)_

_\]_

  

\[

m_i = \exp(\mu_i^{lat})

\]

  

\[

S_u(t \mid i) = 1 - \Phi\!\left(\frac{\log t - \mu_i^{lat}}{\sigma__{LN}}\right)_

_\]_

  

\[

f_u(t \mid i) = \frac{1}{t \, \sigma__{LN}} \, \phi\!\left(\frac{\log t - \mu_i^{lat}}{\sigma__{LN}}\right)

\]

  

---

  

## ****7.4 Log-logistic****

  

\[

k__{LL} = \exp(-\psi__{LL})

\]

  

\[

\lambda_i = m_i = \exp(\mu_i^{lat})

\]

  

\[

S_u(t \mid i) = \frac{1}{1 + (t / \lambda_i)^{k__{LL}}}_

_\]_

  

\[

f_u(t \mid i) = \frac{k__{LL}}{\lambda_i} \left(\frac{t}{\lambda_i}\right)^{k__{LL}-1} \left[1 + \left(\frac{t}{\lambda_i}\right)^{k_{LL}}\right]^{-2}

\]

  

---

  

## ****7.5 Family-specific priors****

  

### ****Exponential****

  

No shape parameter.

  

### ****Weibull****

  

\[

\rho_W \sim N(0, 0.35^2)

\]

  

### ****Log-normal****

  

\[

\log \sigma__{LN} \sim N(0, 0.50^2)_

_\]_

  

### ****Log-logistic****

  

\[

\psi__{LL} \sim N(0, 0.50^2)_

_\]_

  

### ****Why keep these four families?****

  

Because the main goal of Stage 8 is ****not**** to force one single tail story, but to propagate uncertainty across plausible parametric latency shapes. The four families cover constant, monotone, and unimodal hazard patterns, and follow-up-limited data can look similar in the observed region while diverging sharply in the tail.

# ****8. Likelihood**** 🧮

  

\[

S__{\text{pop}, i}(t) = (1 - \pi_i) + \pi_i S_u(t \mid i)_

_\]_

  

\[

R_i(t) = 1 - S__{\text{pop}, i}(t) = \pi_i \{1 - S_u(t \mid i)\}_

_\]_

  

The subject-level likelihood contribution is

  

\[

L_i = \left[\pi_i f_u(T_i \mid i)\right]^{\delta_i} \left[(1 - \pi_i) + \pi_i S_u(T_i \mid i)\right]^{1 - \delta_i}

\]

  

### ****Stan implementation: always use log-space****

  

If \(\delta_i = 1\),

  

\[

\ell_i = \log \pi_i + \log f_u(T_i \mid i)

\]

  

If \(\delta_i = 0\),

  

\[

\ell_i = \log\!\left[(1 - \pi_i) + \pi_i S_u(T_i \mid i)\right] = \operatorname{log\_sum\_exp}\!\big(\log(1 - \pi_i),\ \log \pi_i + \log S_u(T_i \mid i)\big)

\]

# ****10. Prior predictive checks, diagnostics, and Stage-8 admissibility**** 🧪

  

## ****10.1 Prior predictive checks****

  

This Stage 8 block does ****descriptive degeneracy checking****, not external benchmark calibration.

  

For each model, save prior-predictive summaries such as:

  

- cohort mean \(\pi_i\)

- cohort mean risk at 1, 2, 5, and 10 years

- \(LNTF = n^{-1} \sum_i (1 - \pi_i)\)

- cohort median of \(m_i\)

- optionally, representative prior and posterior hazard shapes

  

### ****Automatic fail flags****

  

Flag the fit as prior-degenerate if almost all prior draws imply:

  

1. \(\pi_i \approx 0\) for nearly everyone,

2. \(\pi_i \approx 1\) for nearly everyone,

3. near-zero median susceptible times for almost everyone,

4. implausibly huge susceptible times with effectively flat risk over all supported horizons.

  

---

  

## ****10.2 MCMC diagnostics****

  

A Bayesian fit is ****not admissible**** unless all of the following are acceptable:

  

- divergences = 0

- \(\hat R < 1.01\)

- bulk and tail ESS adequate

- no serious mixing failure across chains

- posterior predictive behavior not grossly contradictory to the observed region

- no prior/posterior degeneracy

  

If using Stan, also save:

  

- trace plots

- effective sample sizes

- divergences

- chain mixing summaries

- any warnings about tree depth / energy behavior

  

---

  

## ****10.3 Internal Bayesian comparison rule****

  

Stage 8 should ****not**** force a single Bayesian “winner”.

  

Instead:

  

1. exclude non-admissible fits,

2. retain ****all admissible Bayesian fits****,

3. save supportive internal criteria such as:

   - pointwise log-likelihood summaries,

   - WAIC / LOOIC,

   - posterior predictive adequacy,

   - hazard / tail plausibility,

4. export all admissible fits to the later unified comparison engine.

  

### ****Interpretive rule****

  

Within Stage 8, internal Bayesian fit criteria are ****supportive only****.  

They do ****not**** override the project’s main scientific target, which is the comparison of risk shift, false-positive burden, and clinical utility on the common horizon/threshold grid.

# ****12. Interpretation rules**** 🎯

  

## ****12.1 Supported-horizon hierarchy****

  

The Bayesian model will still generate posterior trajectories from ****1 to 10 years****, but the inferential weight is ****not uniform****.

  

### ****PNU****

  

- ****1 year****: primary

- ****2 years****: supportive / near-boundary

- ****5 years****: projection

- ****10 years****: projection

  

### ****SNU****

  

- ****1 year, 2 years****: primary

- ****5 years****: secondary

- ****10 years****: projection

  

### ****merged****

  

- ****1 year, 2 years****: primary

- ****5 years****: secondary

- ****10 years****: projection

  

This means:

  

- long-tail behavior ****must still be shown and discussed****, because it is directly relevant to the question of a possible non-transition subgroup,

- but the paper’s main inferential claims should remain anchored to the best-supported horizons.

  

---

  

## ****12.2 Coefficient interpretation****

  

If

  

\[

\gamma__{am} \neq 0_

_\]_

  

then the age effect in latency differs by sex.

  

If

  

\[

\gamma_s \neq 0

\]

  

then PNU and SNU differ in the ****timing of transition among susceptible subjects****.

  

\[

\exp(\gamma_s)

\]

  

can be read as an approximate ****SNU vs PNU median time ratio**** in the susceptible subgroup.

  

If

  

\[

\beta_s^{inc} \neq 0

\]

  

then, for the supplementary merged incidence-site branch, PNU and SNU differ in the ****susceptible probability itself****, even after the shared age-band × sex structure.

  

---

  

## ****12.3 Site-effect interpretation****

  

Any site effect should be interpreted as a proxy for broader ****treatment / care-pathway / context**** differences, not as a clean treatment effect, because direct treatment-administration data are not presently available in the workflow.

  

---

  

## ****12.4 Tail interpretation****

  

Late-horizon divergence across Bayesian latency families is part of the scientific story, not just nuisance.  

However, it should be discussed explicitly as a mix of:

  

- possible cure-like heterogeneity,

- timing differences,

- follow-up immaturity,

- tail extrapolation uncertainty.

  

Hazard shape plausibility and uncertainty “fanning out” matter here, especially when the observed region is short relative to the requested horizon.
